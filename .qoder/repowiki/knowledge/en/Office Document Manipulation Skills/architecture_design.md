- **Modular Skill Structure**: Each document type (docx, pdf, pptx, xlsx) is a self-contained skill with a `SKILL.md` entry point and optional `scripts/`, `references/`, and `assets/` directories.
- **XML-Based Editing (DOCX/PPTX)**: DOCX and PPTX skills rely on an unpack-edit-pack workflow. `unpack.py` extracts ZIP archives and pretty-prints XML; `pack.py` condenses XML and re-zips. Validation is enforced via `BaseSchemaValidator` and format-specific validators (e.g., `DOCXSchemaValidator`, `RedliningValidator`) using ISO-IEC29500 XSD schemas.
- **Scripted Workflows**: Complex tasks are delegated to Python scripts. For example, `pptx/scripts/rearrange.py` handles slide duplication and reordering, while `pptx/scripts/inventory.py` and `replace.py` manage structured text replacement. `docx/scripts/comment.py` manages comment boilerplate across multiple XML files.
- **External Tool Integration**: PDF and XLSX skills leverage external libraries and tools. `xlsx/recalc.py` uses LibreOffice headless mode for formula recalculation. PDF skills reference `pypdf`, `pdfplumber`, and `poppler-utils` for extraction and conversion.
- **Validation Layers**: DOCX/PPTX packing includes auto-repair for common issues (e.g., `xml:space='preserve'`, `durableId` constraints) and strict schema validation to prevent corrupt output.
