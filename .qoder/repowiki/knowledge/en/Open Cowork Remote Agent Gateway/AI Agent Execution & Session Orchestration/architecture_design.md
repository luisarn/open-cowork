- **Core Orchestrator**: The `ClaudeAgentRunner` class serves as the primary entry point, bridging the Electron main process with the `@mariozechner/pi-coding-agent` SDK. It manages session lifecycle, including caching (`piSessions`) to preserve context across turns and invalidating caches when runtime signatures (model, cwd, skills) change.
- **Model Resolution Layer**: Delegates model lookup and API routing to `pi-model-resolution.ts`, which interacts with the `pi-ai` registry. It supports synthetic model fallbacks for custom endpoints and applies runtime overrides for provider-specific compatibilities (e.g., Ollama context windows, OpenRouter API downgrades).
- **Sandboxing & Isolation**: Integrates with WSL (Windows) and Lima (macOS) sandboxes via `sandbox-sync` and `lima-sync`. It synchronizes project files and skills into isolated environments before execution and sanitizes output paths to hide real filesystem structures from the AI.
- **Tool Interception & Security**: Implements a permission-gating hook via `agent.setBeforeToolCall` to intercept tool usage. It includes specialized wrappers for the `bash` tool to handle `sudo` password injection securely via stdin and enforce default timeouts.
- **Event Streaming & Parsing**: Subscribes to SDK events to stream deltas (text, thinking, tool calls) to the renderer. It uses `ThinkTagStreamParser` to separate reasoning content from final answers in streaming responses and employs a `LoopGuard` to detect and abort runaway tool-call patterns.
